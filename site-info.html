Bootstrap Features, Custom Code & Hosting

Overall layout (index.html)

The portfolio is a single-page site built with the Bootstrap grid system. Sections are organised as <section> blocks for About, Education, Experience, Projects, Skills & Languages, Publications, Hobbies and Contact. The <body> tag uses data-bs-spy="scroll" and data-bs-target="#mainNavbar" so Bootstrap Scrollspy can highlight the active nav link while scrolling.

Bootstrap components used

  • Navbar – A fixed-top, responsive navbar (#mainNavbar) with brand, collapsible menu and a “Grading” pill link to this page.
  • Grid system – .container, .row and .col-* classes used to lay out the hero, cards, skills, tables and contact section across desktop and mobile.
  • Cards – .card components are used for the About, Education entries, Experience accordion bodies, Projects, Skills, Hobbies and Contact panels, with custom “glass” styling applied via CSS.
  • Buttons & button groups – Standard Bootstrap buttons (.btn, .btn-primary, .btn-outline-secondary) plus a .btn-group in the Education section to filter the modules table by year.
  • Badges – Skill tags are implemented as customised Bootstrap badges (.badge with extra .skill-pill classes) for languages, libraries, tools and core skills.
  • Accordion – Two separate accordions (#experienceAccordion and #publicationAccordion) are used for Experience and Publications. Their expand/collapse behaviour is handled by Bootstrap’s JS.
  • Carousels – Two carousels (#carsCarousel and #hikingCarousel) display hobby images with previous/next controls and multiple slides.
  • Table & table-responsive – A responsive, centred table of key modules and grades inside .table-responsive, styled to match the dark / glass theme.
  • Forms – The Contact section uses Bootstrap form controls (.form-control, .form-label) and layout for the name, email and message fields.
  • Alerts – A hidden Bootstrap .alert (#contactAlert) is shown with .alert-success or .alert-danger classes after contact-form validation.
  • Modal – The “Open Contact Modal” button uses a Bootstrap modal (#contactModal) to show phone, email and LinkedIn details.
  • Utility classes – Bootstrap spacing, flexbox, alignment and text utilities (e.g. .mb-3, .d-flex, .align-items-center, .text-center, .visually-hidden) are used throughout to control layout and accessibility.

Custom CSS & theming

  • CSS variables and themes – Colour, border and shadow tokens are defined on :root and overridden when body[data-theme="day"] is present, providing a light and dark theme without changing the HTML.
  • Glassmorphism & gradients – The background uses layered radial gradients and a fixed overlay div to create a soft glow. Cards, the navbar and the hero panel use semi-transparent backgrounds, borders, blur and box-shadow for a glass effect.
  • Skill pills & badges – Custom .skill-pill-* classes colour-code different skill categories (languages, libraries, tools, core skills) on top of the base Bootstrap badge styling.
  • Animations & reveal – Sections with .reveal start transparent and translated, then transition to .reveal-visible when scrolled into view, giving a smooth section entrance animation.
  • Hero metrics & layout – The hero section uses a custom “hero grid” card, metric tiles and a gradient headshot frame.

Custom JavaScript features

  • Smooth scrolling – The “Contact Me” button (.js-scroll) and all navbar links that point to section IDs use scrollIntoView / window.scrollTo with an offset for the fixed navbar, giving smooth scrolling between sections.
  • Back-to-top button – A fixed back-to-top button (#backToTop) appears after scrolling down the page and scrolls smoothly back to the top when clicked.
  • Typewriter hero tagline – The hero tagline (#typewriter) cycles through three phrases stored in a phrases array. A custom typewriter loop adds and removes characters to simulate typing and deleting.
  • Animated KPI counters – Metric tiles in the hero ([data-count]) use an IntersectionObserver to start a count-up animation the first time they enter the viewport. The logic handles plain numbers (e.g. 2), decimals and suffixed values such as 6+, while leaving non-numeric values like 1:1 unchanged.
  • Scroll reveal – All sections marked with .reveal are observed with IntersectionObserver; when they cross the threshold, the script adds .reveal-visible and unobserves them so the animation plays only once.
  • Modules table filter – In the Education section, the Year 2 / Year 3 / Year 4 buttons (.module-filter) toggle .d-none on table rows (.module-row with data-year="...") and update the hint text below the table.
  • Contact form validation & alerts – The contact form is validated on submit using vanilla JS (name required, email must contain “@”). Depending on the result, the hidden alert element (#contactAlert) is shown as either .alert-danger or .alert-success. The form is demo-only and does not send data to a backend.
  • Message character counter – The message textarea (#messageInput) updates a live counter (#messageCounter) on each keypress, showing the current length out of the 500-character limit.
  • Theme toggle with localStorage – The “Switch theme” button (#themeToggle) toggles a data-theme="day" attribute on the <body> element and stores the current choice (tc-theme) in localStorage so the light/dark preference persists across page reloads.
  • Headshot image fallback – If the headshot image fails to load, an error handler adds .d-none to the image to avoid showing a broken-image icon.
  • Bootstrap JS components – The page includes the Bootstrap 5.3 bundle, which provides built-in behaviour for the navbar collapse, accordions, carousels, the contact modal and Scrollspy.
